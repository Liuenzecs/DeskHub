# DeskHub 发布资产模板

最后更新：2026-03-28

本文档提供 DeskHub 发版时可直接复用的素材与文案模板，重点覆盖：

- release 截图 shot list
- GitHub Release 文案模板
- 升级须知模板
- 发版 smoke test 记录模板

正式发版流程仍以 [`RELEASE.md`](./RELEASE.md) 为准；这里更偏向“准备什么”和“怎么写”。

## 1. Release 截图 Shot List

建议每次正式版至少准备 6 张截图，统一使用：

- Windows 主环境
- 同一套浅色主题与真实示例数据
- 1440px 以上宽度主画面
- 不出现敏感路径、私人网址、密钥或账号信息

### 必拍 01：总览页

目标：

- 展示 DeskHub 的整体信息架构与高密度控制台风格

画面建议：

- 左侧主导航完整可见
- 顶部全局搜索触发器与“一键上班模式”按钮可见
- 内容区同时出现最近使用、收藏、工作流区块

截图备注模板：

```text
总览页，展示 DeskHub 作为统一工作台的主界面结构。
```

### 必拍 02：命令面板空查询

目标：

- 展示命令面板是第一入口，而不是附属搜索框

画面建议：

- 打开 `Ctrl+K / Cmd+K`
- 结果区能看到最近命令、收藏条目、快速导航

截图备注模板：

```text
命令面板空查询视图，展示最近命令、收藏与快速导航入口。
```

### 必拍 03：命令面板 scoped 搜索

目标：

- 展示 DeskHub 的高频检索体验

画面建议：

- 使用 `workflow:`、`app:` 或 `action:` 之类 scoped query
- 命中结果里最好包含条目、动作或默认工作流操作

截图备注模板：

```text
命令面板支持 scoped 搜索，可直接定位条目、导航或全局动作。
```

### 必拍 04：资源管理页

目标：

- 展示高密度列表管理能力

画面建议：

- 选择一个类型页，例如“项目”或“脚本”
- 同时展示排序、标签筛选、多选模式或批量工具栏中的至少一种

截图备注模板：

```text
资源管理页支持搜索、筛选、排序与批量管理。
```

### 必拍 05：工作流页

目标：

- 展示 DeskHub 的差异化能力

画面建议：

- 至少一张工作流卡片完整可见
- 步骤链、默认工作流状态或执行入口可见

截图备注模板：

```text
工作流卡片会直接展示步骤链，降低执行前的心理门槛。
```

### 必拍 06：数据工具或录入体验

目标：

- 展示 DeskHub 不是只能“启动”，也能可靠“管理”

可选画面二选一：

- 数据工具：备份、恢复、导入导出、健康检查
- 条目录入：原生路径选择器、类型切换表单、工作流步骤编辑

截图备注模板：

```text
DeskHub 提供正式的数据工具与高频录入能力，适合长期使用。
```

## 5. 使用建议

- 正式版发布前，优先保证“截图、release 文案、升级须知、smoke test 记录”四件事同时齐备。
- 若本版没有 portable zip，就不要在 release 文案中保留 Zip 下载占位，避免误导。
- 若本版改到了 SQLite schema、launcher 或数据工具，请务必补“升级提醒”，不要只写功能点。
