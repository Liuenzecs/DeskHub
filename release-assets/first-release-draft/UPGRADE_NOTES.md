# DeskHub 首版 Release 升级须知素材

最后更新：2026-03-28

适用场景：首版发布或从内部试用版升级时，可直接复用或稍作删改后放进 Release 文案。

## 升级前建议

- 先在“数据工具”中执行一次数据库备份。
- 如果你有默认工作流，请在升级后确认默认工作流仍指向正确条目。
- 如果你依赖脚本、工作流命令或项目启动命令，请检查这些路径和命令是否仍然有效。

## 数据层说明

- DeskHub 运行时正式存储已经统一到 SQLite。
- 数据库位置为 `<appDataDir>/data/deskhub.db`。
- 旧的 `items.json` 不会被自动导入、自动迁移或自动删除。
- 如果你曾在更早的 JSON 试验版里录入过数据，需要自行通过“条目 JSON 导入”进入当前版本。

## 升级后建议检查

- 总览页布局、收藏和最近使用是否正常恢复。
- 默认工作流是否仍然可从顶部“一键上班模式”直接启动。
- 命令面板是否还能搜索到你的常用条目、导航和动作。
- 数据工具中的备份、恢复、导入导出和健康检查是否可正常执行。

## 平台与分发说明

- 当前以 Windows 版本为正式主线。
- installer 是当前默认推荐产物。
- portable zip 仍在评估阶段，还没有被锁定为正式长期支持的对外分发方式。

## 已知兼容性边界

- macOS / Linux 目前更适合内部验证与工程化准备，不应在本轮文案中表述成“已完整适配”。
- 如果本地仍残留旧 JSON 文件，它不会参与当前运行时状态同步。
